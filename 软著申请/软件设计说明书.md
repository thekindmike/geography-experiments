## 版本：V1.0
## 日期：2026年6月

---

## 第一章 引言

### 1.1 编写目的
本软件设计说明书旨在详细描述"AI地理实验教学平台"的软件架构、模块设计、数据结构、接口设计等技术细节，为软件开发、测试、维护提供依据。

### 1.2 范围
本说明书适用于"AI地理实验教学平台"V1.0版本，涵盖软件的总体设计、模块设计、数据库设计、UI设计、接口设计等方面。

### 1.3 术语定义
- **控制变量法**：科学实验方法，通过控制自变量、观察因变量，研究变量间的关系
- **互动模拟器**：在浏览器中模拟地理实验现象的交互式程序
- **响应式设计**：适配不同屏幕尺寸的设计方法
- **Canvas**：HTML5绘图技术，用于绘制图形和动画

## 第二章 总体设计

### 2.1 需求规定

#### 2.1.1 功能需求
1. **实验资源库管理**：收录初高中地理实验18个，每个实验包含完整的实验指导
2. **互动模拟实验**：18个实验全部配备互动模拟器，采用控制变量法
3. **智能搜索与筛选**：支持多维度搜索和分类筛选
4. **教师演示模式**：专为课堂教学设计，大字体、高对比度
5. **响应式设计**：适配手机、平板、电脑等多种设备

#### 2.1.2 性能需求
1. **响应时间**：页面加载时间 < 3秒，互动模拟器响应时间 < 0.5秒
2. **并发用户数**：支持1000+并发用户（静态资源部署）
3. **数据准确性**：实验数据准确无误，模拟现象符合科学原理
4. **兼容性**：支持Chrome 80+, Firefox 75+, Safari 13+, Edge 80+等现代浏览器

#### 2.1.3 安全需求
1. **代码保护**：使用JavaScript混淆技术，防止恶意复制和篡改
2. **版权保护**：添加版权声明，明确版权归属
3. **数据安全**：用户数据仅保存在本地浏览器（LocalStorage），不涉及服务器端存储

### 2.2 运行环境

#### 2.2.1 硬件环境
- **客户端**：普通电脑（CPU 2.0GHz+, 内存 4GB+）或移动设备
- **服务器端**：静态资源部署，无需高性能服务器

#### 2.2.2 软件环境
- **操作系统**：Windows, macOS, Linux, iOS, Android等
- **浏览器**：Chrome 80+, Firefox 75+, Safari 13+, Edge 80+等现代浏览器
- **Web服务器**：任意Web服务器（Apache, Nginx, GitHub Pages等）

### 2.3 架构设计

#### 2.3.1 总体架构
本软件采用 **纯前端架构**，无需后端服务器，所有功能在浏览器中完成。

```
┌─────────────────────────────────────────┐
│           浏览器 (Browser)              │
│  ┌─────────────────────────────────┐  │
│  │      HTML5 + CSS3 + JS         │  │
│  │  ┌─────────────────────────┐   │  │
│  │  │   实验数据 (JSON)      │   │  │
│  │  └─────────────────────────┘   │  │
│  │  ┌─────────────────────────┐   │  │
│  │  │   互动模拟器 (Canvas)  │   │  │
│  │  └─────────────────────────┘   │  │
│  └─────────────────────────────────┘  │
└─────────────────────────────────────────┘
                │
                │ (静态资源请求)
                ▼
┌─────────────────────────────────────────┐
│       Web服务器 (Web Server)           │
│  (GitHub Pages / Netlify / Nginx)    │
└─────────────────────────────────────────┘
```

#### 2.3.2 技术栈
- **前端**：HTML5, CSS3, JavaScript (ES6+)
- **绘图**：Canvas API
- **数据存储**：LocalStorage (浏览器本地存储)
- **构建工具**：无（纯手工编写）
- **部署**：GitHub Pages (静态资源托管)

## 第三章 模块设计

### 3.1 首页模块

#### 3.1.1 功能描述
展示平台特色和推荐实验，引导用户进入各功能模块。

#### 3.1.2 界面设计
- **英雄区**：平台标题、简介、统计数据（初中实验数、高中实验数、互动实验数）
- **特色功能区**：6大特色功能卡片
- **推荐实验区**：精选实验卡片

#### 3.1.3 逻辑设计
1. 页面加载时，从 `experiments-data.js` 读取实验数据
2. 统计初中实验数、高中实验数、互动实验数，动态显示
3. 从推荐实验列表中，选取前6个实验，生成推荐实验卡片

### 3.2 实验列表模块

#### 3.2.1 功能描述
展示初中或高中实验列表，支持分类筛选。

#### 3.2.2 界面设计
- **筛选栏**：按主题分类筛选（如"地球与地图"、"天气与气候"等）
- **实验卡片网格**：展示实验卡片，每张卡片包含实验名称、难度、时长、标签等信息

#### 3.2.3 逻辑设计
1. 页面加载时，根据URL参数（junior或senior），筛选相应实验数据
2. 生成实验卡片，插入到网格中
3. 点击筛选按钮时，重新筛选实验数据，更新实验卡片

### 3.3 实验详情模块

#### 3.3.1 功能描述
展示实验的完整信息，包括实验目的、原理、器材、步骤、数据记录、结果分析、思考题等。

#### 3.3.2 界面设计
- **实验信息区**：名称、难度、时长、评分、标签
- **实验目的区**
- **实验原理区**
- **实验器材区**
- **实验步骤区**
- **数据记录区**（部分实验）
- **结果分析区**
- **思考题区**
- **操作按钮区**：返回按钮、开始实验按钮（如有互动模拟器）

#### 3.3.3 逻辑设计
1. 从URL参数获取实验ID
2. 从 `experiments-data.js` 读取相应实验数据
3. 根据实验数据，动态生成实验详情页面
4. 如该实验有互动模拟器，显示"开始实验"按钮

### 3.4 互动模拟器模块

#### 3.4.1 功能描述
在浏览器中模拟地理实验现象，学生可调节自变量、观察因变量，记录实验数据，分析实验结论。

#### 3.4.2 界面设计
- **控制变量说明区**：说明实验的控制变量、自变量、因变量
- **自变量调节面板**：使用滑块或按钮调节自变量参数
- **因变量观察区**：使用Canvas绘制实验现象，实时显示因变量变化
- **数据记录表面板**：显示已记录的实验数据
- **科学结论分析框**：引导学生分析实验结论

#### 3.4.3 逻辑设计
1. 从URL参数获取实验ID
2. 根据实验ID，调用相应的互动模拟器初始化函数（如 `initLatitudeSimulator()`）
3. 初始化Canvas绘图环境
4. 绑定滑块或按钮的事件监听器，当自变量变化时，重新计算因变量，更新Canvas绘图
5. 点击"记录数据"按钮时，将当前自变量和因变量值添加到数据记录表
6. 点击"重置"按钮时，重置所有参数，清空数据记录表

#### 3.4.4 控制变量法算法
```
// 伪代码
function updateExperiment() {
    // 1. 读取自变量值
    let independentVar = getIndependentVar();

    // 2. 控制其他变量不变
    controlOtherVars();

    // 3. 计算因变量值
    let dependentVar = calculateDependentVar(independentVar);

    // 4. 更新因变量观察区
    updateObservationArea(dependentVar);

    // 5. 可选：记录数据
    if (recordData) {
        addDataRecord(independentVar, dependentVar);
    }
}
```

### 3.5 搜索模块

#### 3.5.1 功能描述
支持按实验名称、关键词、知识点等多维度搜索实验。

#### 3.5.2 界面设计
- **搜索栏**：输入框、搜索按钮、关闭按钮
- **搜索结果下拉列表**：实时显示搜索结果

#### 3.5.3 逻辑设计
1. 点击搜索图标时，显示搜索栏
2. 在输入框中输入关键词时，实时搜索实验数据
3. 搜索算法：遍历实验数据，匹配实验名称、关键词、知识点等字段
4. 将搜索结果动态显示在搜索结果下拉列表中
5. 点击搜索结果时，跳转到相应实验详情页面

### 3.6 教师演示模式模块

#### 3.6.1 功能描述
专为课堂教学设计，大字体、高对比度，方便教师课堂投影展示。

#### 3.6.2 界面设计
- **大字体**：字体大小 × 1.5
- **高对比度**：背景白色，文字黑色，边框加粗
- **简化界面**：隐藏不必要的元素（如搜索栏、移动端菜单等）

#### 3.6.3 逻辑设计
1. 按快捷键 `F11` 进入全屏模式
2. 点击"教师演示模式"按钮时，给 `<body>` 添加 `demo-mode` 类名
3. 在CSS中，定义 `.demo-mode` 的样式（大字体、高对比度等）
4. 点击"退出演示"按钮时，移除 `<body>` 的 `demo-mode` 类名

## 第六章 接口设计

### 6.1 内部接口

#### 6.1.1 实验数据接口
- **功能**：提供实验数据
- **接口**：`experiments_junior`, `experiments_senior`（全局变量）
- **返回值**：实验数据数组

#### 6.1.2 互动模拟器接口
- **功能**：初始化互动模拟器
- **接口**：`initLatitudeSimulator()`, `initRotationSimulator()`, ...（全局函数）
- **参数**：无
- **返回值**：无

#### 6.1.3 页面切换接口
- **功能**：切换页面
- **接口**：`showPage(pageName)`（全局函数）
- **参数**：pageName（字符串，如"home", "junior", "senior"等）
- **返回值**：无

### 6.2 外部接口

#### 6.2.1 浏览器接口
- **LocalStorage**：用于保存用户偏好设置和实验数据记录
- **Canvas API**：用于绘制互动模拟器的实验现象
- **requestAnimationFrame**：用于实现流畅的动画效果

#### 6.2.2 第三方服务接口
- **GitHub Pages API**：用于自动部署（通过git push触发）
- **Netlify API**：用于自动部署（可选）
